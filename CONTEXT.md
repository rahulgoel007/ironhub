# IronHub

IronHub is the catalog and account surface for IronClaw skills and tools.

## Language

**User Profile**:
The signed-in person's visible account record in IronHub.
_Avoid_: Account profile, wallet profile

**NEAR Account**:
A NEAR wallet identity associated with a signed-in user.
_Avoid_: User, profile, account

**Account Page**:
The `/account` page where a person signs in and views their User Profile.
_Avoid_: Login page, dashboard

**Sign-In Identity**:
A single provider-specific login identity that owns one User Profile.
_Avoid_: Linked account, merged account

**Catalog**:
The browsable IronHub inventory of skills, tools, and public Iliad entries.
_Avoid_: List, marketplace data

**Marketplace Entry**:
A single skill or tool shown in the Catalog.
_Avoid_: Item, card

**Agent Loadout**:
A selected bundle of skills, tools, and persona settings for an IronClaw agent.
_Avoid_: Agent config, preset

**Agent Installation**:
An IronClaw agent endpoint registered by a User Profile as a target for signed installs.
_Avoid_: Agent profile, agent account, agent config

**Shared Install Key**:
A secret generated by an IronClaw agent and stored by IronHub to sign install intents for that agent.
_Avoid_: API key, access token, hub key

**Key Fingerprint**:
A non-secret display hint derived from a Shared Install Key so users can identify which key is active.
_Avoid_: Masked key, public key

**Install Intent**:
A short-lived signed request from IronHub asking an Agent Installation to install a Marketplace Entry.
_Avoid_: Install link, deep link, redirect token

**Install Intent Record**:
IronHub's audit record of an Install Intent it issued.
_Avoid_: Install session, install event

## Relationships

- A **User Profile** can have zero or one **NEAR Account**.
- A **User Profile** belongs to exactly one **Sign-In Identity**.
- A **Sign-In Identity** creates exactly one **User Profile**.
- An **Account Page** shows exactly one signed-in **User Profile** when a session exists.
- A **Catalog** contains one or more **Marketplace Entries**.
- An **Agent Loadout** can include zero or more **Marketplace Entries**.
- A **User Profile** can register one or more **Agent Installations**.
- An **Agent Installation** belongs to exactly one **User Profile**.
- An **Agent Installation** has exactly one active **Shared Install Key**.
- A **Key Fingerprint** identifies a **Shared Install Key** without revealing it.
- An **Install Intent** targets exactly one **Agent Installation** and one **Marketplace Entry**.
- An **Install Intent Record** belongs to exactly one **Install Intent**.

## Example dialogue

> **Dev:** "When a person signs in with NEAR, do we show the wallet as the User Profile?"
> **Domain expert:** "No. The User Profile is the IronHub account record; the NEAR Account is the Sign-In Identity for that profile."
> **Dev:** "Can an Agent Loadout include any Marketplace Entry from the Catalog?"
> **Domain expert:** "Yes. The Catalog is the source list; the Agent Loadout is the user's selected bundle."
> **Dev:** "Does a User Profile point to one IronClaw endpoint?"
> **Domain expert:** "No. A User Profile can register multiple Agent Installations, with one used as the default install target."
> **Dev:** "Does IronHub create the Shared Install Key?"
> **Domain expert:** "No. IronClaw creates the Shared Install Key, and IronHub stores it only to sign install intents."
> **Dev:** "Can the Account Page show the Shared Install Key later?"
> **Domain expert:** "No. It can show the Key Fingerprint, but the Shared Install Key is write-only after save."
> **Dev:** "What happens if a user clicks install before registering an Agent Installation?"
> **Domain expert:** "IronHub sends them to the Account Page to register an Agent Installation, then resumes the Install Intent."
> **Dev:** "Does IronHub need to remember every Install Intent?"
> **Domain expert:** "Yes, as an Install Intent Record for support and debugging, without storing replayable signature material."

## Flagged ambiguities

- "account" can mean the signed-in **User Profile**, a **NEAR Account**, or the **Account Page**; resolved by using these exact terms.
- "account linking" is not part of IronHub auth; each **Sign-In Identity** owns a separate **User Profile**.
